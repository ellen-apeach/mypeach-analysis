채팅은 사실상 변경 이력 관리 도구로 쓰이고 있습니다. 고객이 채팅에서 "이건 유지", "이건 변경"을 계속 남기지만, 채팅은 흐름형이라 나중에 최신 확정 내용을 찾기 어렵습니다. 이것이 반복 재문의의 핵심 원인입니다.

🔴 고위험 신호

누수/하자 의심, 소음/민원 발생, 일정 지연 — 즉각적인 매니저 개입이 필요한 신호. 방치 시 신뢰 이슈로 전환 가능.

🟡 주의 필요 신호

마감 퀄리티 반복 지적, 추가비용 투명성 요구, 응답 지연 누적 — 불만이 고위험으로 전환되기 전 선제 대응 필요.

Product Opportunity

채팅 데이터가 말하는 제품 기회

채팅을 더 잘 만드는 것이 아니라, 채팅 속 정보를 구조화하는 방향입니다.

1순위 고객 확인 필요함

채팅에 흩어진 고객 액션을 분리해서 보여주기. "선택만 해주시면 돼요" 톤 — 업무가 아닌 서비스 요청처럼 느껴지게.

매니저 리마인드 부담 감소 · 의사결정 지연 감소

2순위 공간별 결정사항 보드

자재/마감/옵션을 공간 단위로 정리. 상태값: 확정 → 현장전달 → 시공반영 → 변경됨.

선택 누락/오해 감소 · 재확인 채팅 감소

3순위 변경사항 타임라인

자재/일정/비용 변경 이력과 후속 영향(비용·일정·현장 전달)을 연결해서 보여주기.

분쟁 예방 · 신뢰 리스크 감소

4순위 AS/하자 이슈 트래커

완료 후 품질 이슈를 채팅에서 분리해 상태값으로 관리. 접수 → 확인 중 → 처리 완료.

"말했는데 처리됐나?" 불안 해소

5순위 일정 영향 표시

변경이 전체 완료일/입주일에 영향을 주는지 판단 정보 제공. 단순 캘린더가 아닌 영향도 표시.

입주 불안 완화 · 반복 일정 문의 감소

6순위 매니저 응대 초안

반복 상황별 초안 자동 생성. 완전 자동 발송 X — 초안 생성 → 매니저 수정 → 발송 플로우.

응대 품질 균일화 · 매니저 시간 절감

// 최종 결론

고객이 채팅을 많이 하는 이유는 직접 관리하고 싶어서가 아니라, 맡긴 일이 제대로 관리되고 있는지 확신이 부족했기 때문이다. MyPeach의 핵심 기회는 채팅을 더 편하게 만드는 것이 아니라, 채팅 속 정보를 고객이 안심할 수 있는 프로젝트 상태 정보로 바꾸는 것이다.
